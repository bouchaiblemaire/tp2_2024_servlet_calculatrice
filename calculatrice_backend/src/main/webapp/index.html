🧮 Corrigé du de la calculatrice back-end

http://ip_serveur:port/calculatrice_etape_1/calculate?operation=xx&operande1=yy&operande2=zz

Paramètres de l’URL

  • ip_serveur : adresse IP du serveur
  • port : port d'écoute du serveur (ex. 8080)
  • xx : opération (add, sub, mul, div)
  • yy : valeur de l'opérande 1
  • zz : valeur de l'opérande 2

Exemple d’appel

http://ip_serveur:port/calculatrice_etape_1/calculate?operation=add&operande1=5&operande2=7

⚠️ Avertissement pédagogique

Ce code utilise volontairement des anti-patterns.

Il n’a pas un bon “good smell”.

Ce code doit être refactorisé pour respecter les principes du “good smell code”.

🚫 Anti-patterns courants :

  • Spaghetti Code — absence de structure claire, dépendances enchevêtrées
  • God Object — une classe fait tout, sans séparation des responsabilités
  • Copy-Paste Programming — duplication du code au lieu de factorisation
  • Magic Numbers / Strings — valeurs codées en dur sans signification explicite
  • Shotgun Surgery — une petite modification implique des changements partout
  • Golden Hammer — utilisation abusive d’un même outil ou pattern pour tout
  • Premature Optimization — optimisation trop tôt, au détriment de la lisibilité
  • Feature Envy — une méthode accède trop aux données d’une autre classe
  • Lazy Class — classe inutile qui n’a pas de réelle responsabilité
  • Dead Code — code jamais exécuté, obsolète ou inutile
